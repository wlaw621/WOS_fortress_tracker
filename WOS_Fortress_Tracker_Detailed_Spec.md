# WOS 요새쟁탈 명단작성 PRO 시스템 상세서

본 문서는 **1953 GOM 연맹**의 요새전 운영 효율화를 위해 개발된 명단작성 도구의 최종 기술 명세입니다.

---

## 1. 시스템 아키텍처
- **Frontend**: React (Vite) + Tailwind CSS
- **Backend**: Google Apps Script (GAS)
- **Database**: Google Sheets (참여자목록, 참여현황 탭)
- **AI Engine**: Google Gemini API (`gemini-2.5-flash`)

---

## 2. 핵심 기술 명세

### 2.1 AI OCR 엔진 (`gemini-2.5-flash`)
- **모델 확정**: 최신 `gemini-2.5-flash` 모델을 통한 고속/고성능 이미지 분석.
- **지능형 프롬프트 (v1.3.4)**: 
  - "이미지에서 연맹원 닉네임을 모두 추출하여 JSON 형식으로 반환해줘. 결과값은 반드시 {"names": ["닉네임1", "닉네임2"]} 형태로 받도록 강제" 하는 지시어 적용.
- **데이터 정제 알고리즘 (Algorithm Specification)**:
  - **특수문자 제거**: `cleanStr` 함수를 통해 한글, 영문, 숫자 외의 이모지 및 특수문자를 제거하여 인식 오류 및 중복 방지.
  - **유사도 매칭 (v1.3.4 고도화)**: 
    - (1) 완전 일치 및 포함 관계 분석 (Inclusion) - 영문 대문자 정규화 적용.
    - (2) **접두사 매칭 (Prefix Matching)**: 앞 2글자 이상 일치 시 동일 유저로 판단 (예: `ILYSM...` 매칭 최적화).
  - **이름 표시 정책**: 매칭 성공 시 OCR 결과 대신 **마스터 명단에 등록된 공식 이름**을 리스트에 표시. 신규 유저의 경우에도 특수문자를 제거한 정제된 이름으로 표시.
  - **역할 우선순위**: 인식 결과와 상관없이 마스터 명단에 등록된 정보(운영진/본캐/부캐)를 최우선 적용.
  - **중복 제거**: 동일 유저가 여러 표기법으로 인식되더라도 마스터 명단 기준으로 자동 병합(Deduplication).

### 2.2 실시간 데이터 동기화 (GAS)
- **참여자목록(Master)**: 마스터 명단 데이터 실시간 로드 및 개별 셀 업데이트 지원. (A:이름, B:등급, C:분류 매핑 준수)
- **참여현황(Status) (v1.3.4)**: 
  - **일괄 처리(Batch Update)**: 한 줄씩 기록하는 방식에서 전체 데이터를 필터링 후 한꺼번에 쓰는 방식으로 개선하여 안정성 및 속도 극대화.
  - **시간대 독립성**: 시간대가 다르면 동일 인물이라도 중복으로 간주하지 않고 별도 기록 보장.
- **CORS 대응**: Simple Request 방식을 사용하여 브라우저 정책 준수.

### 2.3 UI/UX 디자인
- **테마**: 설원(Blizzard) 테마 기반의 화이트/블루 톤 적용.
- **저장 시스템**: 자동 저장 로직 외에 **[시트에 저장]** 버튼을 수동 추가하여 사용자에게 시각적 피드백 제공.
- **모달 시스템**: 유저 추가 시 현대적인 글래스모피즘(Glassmorphism) 스타일의 팝업 제공.
- **버전 관리**: 배포 상태 확인을 위해 메인 및 관리자 화면에 버전 표시(`v1.3.4`) 적용.

## 3. 기능 명세 히스토리

### 3.1 명단 관리 (Admin)
- **유저 추가**: 이름, 등급, 분류를 선택하여 시트 최하단에 즉시 추가.
- **유저 수정**: 리스트 내에서 직접 정보 수정 시 실시간 반영.
- **유저 삭제**: 유저명을 포함한 확인 팝업 후 안전하게 시트 데이터 삭제.

### 3.2 참여 현황 (Main)
- **시간대별 관리**: 12시/18시/21시(22시) 독립 세션 운영.
- **명단 정렬**: 운영진(1) > 본캐(2) > 부캐(3) 순서 적용 후 가나다(ㄱ-ㅎ) 순 정렬.
- **명단 복사**: `1. 닉네임 (역할)` 포맷으로 정렬된 참여 명단을 클립보드에 복사.

## 4. 유지보수 가이드
- **API 키**: `.env` 파일의 `VITE_GEMINI_API_KEY`를 통해 관리.
- **GAS 주소**: `App.jsx` 내 `CONFIG.GAS_URL`에 웹 앱 URL 설정.
- **데이터 구조**: 구글 시트의 `참여자목록` 탭 컬럼 순서(이름, 등급, 분류) 엄수.
- **복구 대책**: 시스템 안정화를 위해 `src/App_v1.2.5.bak.jsx` 백업 파일 운용.
