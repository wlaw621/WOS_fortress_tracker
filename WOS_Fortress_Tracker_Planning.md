# WOS 요새전 참여 관리 시스템 기획안 (GAS 최적화 버전)

## 1. 프로젝트 개요
- **프로젝트명**: WOS Fortress Tracker ([wageulwageul.com](https://www.google.com/search?q=wageulwageul.com))
- **목적**: "Whiteout Survival" 요새전 및 성 채우기 참여 인원을 스크린샷 OCR 분석을 통해 자동 기록 및 관리
- **핵심 변경사항**: 별도의 서버 구축 대신 Google Sheets를 데이터베이스로, GAS를 백엔드 API로 활용하여 유지보수 비용 제로화 및 실시간 동기화 구현

---

## 2. 시스템 아키텍처
기존 기획의 복잡한 서버 구조를 제거하고 구글 생태계를 적극 활용합니다.
- **Frontend**: Vercel 기반의 React 웹 페이지 (Blue-Blizzard 테마)
- **Backend/API**: Google Apps Script (GAS)
- **Database**: Google Sheets (마스터 리스트 및 참여 로그 저장)
- **AI Engine**: Google Gemini API (이미지 분석 및 닉네임 추출)

---

## 3. 주요 기능 수정 내용

### A. 데이터 관리 및 저장 (GAS 연동)
- **실시간 동기화**: fetch API를 사용하여 구글 시트의 데이터를 실시간으로 읽어오고 기록합니다.
- **마스터 리스트 대조**: 시트의 `참여자목록` 탭에 등록된 닉네임, 등급, 분류(운영진/본캐/부캐) 정보를 기반으로 스캔 결과를 자동 분류합니다.
- **참여 기록 자동화**: 스캔이 완료되면 `참여현황` 탭에 타임스탬프와 함께 명단이 즉시 추가됩니다.

### B. OCR 및 AI 분석 (Gemini API)
- **멀티 이미지 스캔**: 집결(Rally) 및 솔로(Solo) 탭 스크린샷을 동시에 분석합니다.
- **지능형 추출**: `[GOM]`과 같은 연맹 태그를 AI가 스스로 판단하여 제거하고 순수 닉네임만 추출합니다.
- **미등록자 식별**: 마스터 리스트에 없는 유저는 '미등록'으로 별도 표시하여 명단 누락을 방지합니다.

### C. UI/UX 개선
- **디자인**: 사용자 선호도에 맞춘 화이트/블루 톤의 빙원(Blizzard) 테마 적용
- **접근성**: ID/비밀번호 로그인 없이 간단한 예약제 기반 식별 시스템 사용
- **편의성**: 분석 결과를 한 번에 복사할 수 있는 '명단 복사' 버튼 및 모바일 최적화 레이아웃

---

## 4. 데이터 구조 (Google Sheets)

| 탭 이름 | 주요 필드 (Column) | 비고 |
| :--- | :--- | :--- |
| **참여자목록** | 이름, 등급, 분류 (운영진/본캐/부캐) | 마스터 데이터 |
| **참여현황** | 날짜/시간, 닉네임, 등급, 분류 | 분석 완료 시 자동 기록 |

---

## 5. 향후 확장 계획
- **대시보드 구현**: 구글 시트의 데이터를 활용하여 월간 참여도 및 연맹원 활동량 차트(Radar/Timeline) 시각화
- **자동 수정 루틴**: 웹 UI의 '명단 관리' 탭에서 수정한 내용이 구글 시트 마스터 데이터에 즉시 업데이트되는 양방향 통신 강화
